Executing the Code

After understanding the code to some extent, I executed it and got the following error:

UnicodeDecodeError: 'utf-8' codec can't decode byte 0xc4 in position 16: invalid continuation byte

What it meant was that the training set that I was using contained such mails which python was not able to decode using UTF-8.

This could simply imply that the mails were not encoded using UTF-8 in the first place. But searching with the error message did not resulted in much fruitful solutions so I consulted my head. According to him, it was completely ok for the emails to be encoded in any format. It is our code that should be flexible enough to deal with all the formats. We need to ensure maximum customer convenience as a software company. And yeah, he had a point.
